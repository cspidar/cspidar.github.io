    움직이는 이미지를 만들어 문서에 인터렉티브함과 가독성을 높이고자 함.

    1. 왜 ScreenToGif?#

    오픈소스. 짜증나는 광고, 워터마크, 분량제한 없이 사용할 수 있다.
    오픈 소스. 녹화 범위 지정 가능. 마우스 따라다니기 기능 있음. 설정 항목이 다양하고 방법이 직관적. gif, avi 등 다양한 형식 제공
    마우스 위치 강조나 클릭 효과가 없지만 (못 찾은건지..) 고정창 마우스 따라가기 기능이 더 효율적이고 시인성도 높다고 판단.
    https://www.screentogif.com/

    비교 대상

    • CamStudio
      오픈소스. 마우스 위치 강조나 클릭 효과 있음.
      출력 양식 부족, 부가 기능 부족
      소개 사이트에서의 인터페이스는 처참하지만..
      실제 윈도우10 환경에서 사용해보면 봐줄만 하다. 윈도우 기본 인터페이스를 사용해서 그런듯.

    • ShareX
      https://getsharex.com/
      오픈소스. 추천 사이트에서 핫함, 자체 서비스에 쉽게 업로드하는것에 집중된 느낌?
      영역 설정 기능이 없다.
      녹화 관련 설정은 이게 전부.

      ShareX 화면 녹화 설정 항목

      화면 영상 녹화보다는 이미지 캡처 및 편집에 집중한 느낌, 하지만 이미지 캡처 관련해서는 다양한 양식을 지원하고 문서도 훌륭하게 구성되어 있다.

      https://getsharex.com/image-effects/

    • Free Cam
      무료 인듯 하면서 유료.
      무료 - 유료 기능 차이
      https://www.freescreenrecording.com/cam-pro

    이외에 더 테스트해보고 더좋으면 메인으로, 아니면 여기 리스트에 추가할 예정입니다.

    내가 필요로 한 + ScreenToGif에서 만족하는 기능 리스트

    • 캡처 영역 설정 기능

    • 마우스 커서 강조, 클릭 표시 기능 → 마우스 따라다니기 기능

    • 녹화 시작 중지 단축키

    • 무료이면서 광고 워터마크 분량제한이 없을것

    • 결과물에 대한 자유로운 사용 가능

    • 등등

    여러 이유가 있겠지만 현재로써는 가장 나은 옵션인듯
